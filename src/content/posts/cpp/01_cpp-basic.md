# \[ C++ 기초 ]
## C++ 뼈대와 입출력 구조 이해

### C++의 정체성

C++ 코드를 처음 보면, 어떤 부분은 C 언어처럼 위에서 아래로 절차적으로 실행되는 것 같고, 어떤 부분은 클래스와 객체를 사용하는 객체지향의 모습을 띠고 있습니다. 이는 C++이 처음 탄생할 때 'C with Classes(클래스가 추가된 C)'라는 이름으로 시작했기 때문입니다.

자바(Java)처럼 모든 함수와 변수가 반드시 클래스 내부에 속해야 하는 엄격한 객체지향 언어와 달리, C++은 **절차지향 프로그래밍**과 **객체지향 프로그래밍**을 개발자가 상황에 맞게 섞어서 쓸 수 있는 '멀티 패러다임'을 지원합니다.

### `int main()`의 진짜 의미

C++ 프로그램은 무조건 `main` 함수에서 시작합니다. 자바스크립트나 파이썬처럼 파일의 첫 줄부터 스크립트가 쭉 실행되는 형태가 아니라, 운영체제(OS)가 프로그램을 실행할 때 가장 먼저 호출하는 **진입점(Entry Point)**이 바로 `main` 함수입니다.

```cpp
int main() {
  // 프로그램 로직
  return 0;
}
```

> 왜 하필 반환형이 `int`일까요? 이 프로그램이 끝났을 때 나를 실행한 **운영체제에게 상태 코드를 보고하기 위해서**입니다.

### 헤더 파일과 `#include <iostrea>`의 동작 방식

화면에 무언가를 출력하려면 가장 먼저 소스 코드 맨 위에 아래와 같은 줄을 적어야 합니다.

```cpp
#include <iostream>
```

Java의 `import`나 Node.js의 `require`, Python의 `import`와 비슷해 보이지만, **내부 동작 방식은 완전히 다릅니다.** 다른 언어들의 `import`가 모듈을 논리적으로 연결하거나 객체 참조를 가져오는 것이라면, C++의 `#include`는 **물리적인 복사 붙여넣기**입니다.

- `#`이 붙은 명령어는 컴파일이 시작되기 전 '전처리기(Preprocessor)'가 처리합니다.
- `<iostream>`이라는 파일(Input/Output Stream을 다루는 표준 라이브러리)의 모든 텍스트 내용을 현재 파일에 그대로 복사해서 붙여넣어 줍니다. (C++20부터는 모듈 개념이 도입되었지만, 아직까지는 이 방식이 표준처럼 쓰입니다.)

### `std::cout`과 `std::cin`: 인스턴스와 스트림 입출력 흐름

C++에서 콘솔 입출력을 담당하는 `std::cout`과 `std::cin`은 단순한 함수가 아니라, `iostream` 표준 라이브러리가 미리 만들어 둔 **인스턴스(객체)**입니다. 

```cpp
int age;
std::cout << "나이를 입력하세요: ";
std::cin >> age;
std::cout << "당신의 나이는 " << age << std::endl;
```

- **스트림(Stream)과 인스턴스:** 데이터가 강물처럼 흘러가는 통로(스트림)를 객체로 구현한 것입니다. `std::cout`은 모니터로 향하는 출력 스트림 인스턴스, `std::cin`은 키보드의 입력 스트림 인스턴스입니다.
- **연산자 오버로딩 `<<`, `>>`:** 데이터를 흘려보내는 연산자입니다. `"나이를 입력하세요"`라는 문자열을 `std::cout` 객체 쪽으로 밀어 넣고(`<<`), `std::cin` 객체에서 들어온 데이터를 `age` 변수 쪽으로 빼냅니다(`>>`). 객체지향의 특징을 활용해 직관적인 입출력 흐름을 만들어냅니다.
- **타입 안전성 (Type Safety):** `printf`처럼 `%d`, `%s` 같은 포맷을 맞출 필요 없이, 입력/출력 객체가 변수의 타입(`int`, `string` 등)을 알아서 추론하여 안전하게 입출력해 줍니다.

### 네임스페이스(`namespace`)와 `using namespace std;`의 함정

C++의 표준 라이브러리 기능들은 모두 `std` (Standard의 약자)라는 네임스페이스 안에 들어있습니다.

매번 `std::`를 붙이기 귀찮아서 초보자 튜토리얼을 보면 대부분 아래와 같은 코드가 등장합니다.

```cpp
using namespace std;
cout << "Hello!" << endl;
```

하지만 **실무나 규모가 있는 프로젝트에서는 `using namespace std;`를 전역 공간에 쓰는 것을 강력하게 지양합니다.** 이유는 **이름 충돌(Name Collision)** 때문입니다. C++ 표준 라이브러리에는 수많은 함수와 객체가 존재합니다(`count`, `distance`, `max` 등). 무심코 `std` 공간을 전부 열어버리면, 내가 직접 만든 변수나 함수 이름과 표준 라이브러리의 이름이 겹쳐서 원인을 찾기 힘든 컴파일 에러가 발생할 수 있습니다.

**해결책:** 귀찮더라도 명시적으로 `std::cout`, `std::cin`을 적어주거나, 꼭 필요하다면 함수 내부 등 좁은 스코프 안에서만 사용하는 것이 현대 C++의 좋은 습관입니다.

## 변수와 엄격한 자료형 (Data Types)

프로그래밍에서 변수는 데이터를 담는 상자입니다. 하지만 C++에서의 상자는 자바스크립트의 `let`이나 파이썬의 변수처럼 아무거나 다 담을 수 있는 마법의 상자가 아닙니다. C++의 자료형은 철저하게 **"메모리"**와 직결되어 있습니다.

### 정적 타입 시스템

```cpp
// JavaScript나 Python의 유연함 (C++에서는 불가능)
// let data = 10;
// data = "Hello";

// C++의 엄격함
int data = 10;
// data = "Hello"; // 컴파일 에러 발생
```

C++은 코드를 기계어로 번역하는 **컴파일 타임(Compile-time)**에 모든 변수의 크기와 형태가 결정되어야 합니다. 그래야 프로그램이 실행될 때 타입 검사로 인한 지연 시간 없이, 메모리를 정확히 계산하고 최고 속도로 동작할 수 있기 때문입니다.

### 기본 원시 타입

| 자료형 (Type) | 일반적인 크기 | 설명 및 표현 범위                                                                         | 예시                            |
| ------------- | ------------- | ----------------------------------------------------------------------------------------- | ------------------------------- |
| `bool`        | 1 byte        | 논리형: 참(`true`) 또는 거짓(`false`)                                                     | `bool isDone = true;`           |
| `char`        | 1 byte        | 문자형: 알파벳이나 기호 하나를 담으며, ASCII 코드로 저장                                  | `char grade = 'A';`             |
| `int`         | 4 byte        | 기본 정수형: 약 -21억 ~ 21억까지의 숫자를 표현                                            | `int age = 28;`                 |
| `long long`   | 8 byte        | 확장 정수형: 21억을 넘어가는 큰 숫자를 다룸                                               | `long long views = 3000000000;` |
| `double`      | 8 byte        | 배정밀도 실수형: `float`보다 2배 더 정밀한 실수형으로, <br>소수점 아래 약 15자리까지 유효 | `double exactPi = 3.14159265;`  |

> 주의할 점: C++ 원시 타입의 크기는 운영체제(32bit/64bit)나 컴파일러에 따라 약간씩 달라질 수 있습니다.
> 위 표는 64비트 환경에서 일반적으로 쓰이는 기준입니다.  

> [!TIP]
**팁: 명시적 부호화(`unsigned`)**  
> 정수형의 경우 앞에 `unsigned`를 붙여 의도적으로 "음수가 절대 나올 수 없음"을 명시할 수 있습니다. (예: `unsigned int age = 28;`) 이 경우 음수를 표현하지 않는 대신, 양수 표현 범위가 2배로 늘어납니다.

모던 C++(C++11 이후)에서는 여러 복잡한 자료형을 대신해 `auto`라는 키워드가 도입되어 `auto num = 10;`처럼 타입을 컴파일러가 추론하게 만들 수도 있습니다. 하지만 이는 자바스크립트의 `let`처럼 실행 중인 타입이 변할 수 있다는 뜻이 아니라, 초기화되는 값을 보고 컴파일러가 `int`로 **고정**해주는 것입니다. 자료형 자체가 바뀌는 것은 허용되지 않습니다.

### C 스타일 문자열 vs `std::string`

C++에서 문자열을 다룰 때는 두 가지 방식을 명확히 구분해야 합니다.

#### 1. C 스타일 문자열 (원시 배열)

C++의 근간이 되는 C언어에서는 사실 '문자열'이라는 독립된 자료형이 없습니다. 대신 문자의 배열(`char[]`)을 사용합니다.

```cpp
char greeting[] = "Hello";
```

이 방식은 메모리를 최소한으로 쓰지만, 배열의 크기를 변경하기 어렵고 길이를 구하거나 문자를 이어 붙이는 작업이 매우 번거롭고 위험합니다.

#### 2. `std::string`

자바의 `String`이나 자바스크립트의 문자열처럼 편하게 쓰고 싶다면, 표준 라이브러리인 `<string>`을 불러와 `std::string` 클래스를 사용하면 됩니다.

```cpp
#include <iostream>
#include <string>

int main() {
  std::string name = "toran";
  std::string greeting = "Hello, " + name + "!";

  std::cout << greeting << " (길이: " << greeting.length() << ")" << std::endl;
  return 0;
}
```

`std::string`은 객체지향적으로 설계되어 있어, 길이를 자동으로 조절해 주고 `.length()`, `.append()` 같은 유용한 메서드들을 제공합니다.

## 연산자와 제어문, 그리고 형 변환

자바나 자바스크립트 같은 언어에 익숙하다면, C++의 제어문과 연산자는 이미 다 알고 있는 것과 다름없습니다. 모두 같은 C 언어의 문법적 뿌리를 공유하고 있기 때문입니다.

하지만 파이썬과 같은 언어와 비교했을 때 주의해야 할 세밀한 차이점과, C++만의 엄격한 형 변환(Casting) 방식을 짚고 넘어갈 필요가 있습니다.

### 연산자 사용 시 주의점: 정수 나눗셈

`+`, `-`, `*`, `%`, `&&`, `||`같은 기본 연산자는 다른 언어와 완전히 동일하게 동작합니다. 다만, 파이썬이나 자바스크립트를 주로 사용하셨다면 **나눗셈(`/`)** 연산에서 흔히 실수를 겪을 수 있습니다.

C++에서는 정수와 정수를 나누면 결과도 무조건 정수(소수점 이하 버림)가 됩니다.

```cpp
int a = 5;
int b = 2;

// JavaScript나 Python에서는 2.5가 나오지만, C++에서는 2가 나옵니다.
int result1 = a / b;

// 소수점 결과를 얻으려면 둘 중 하나 이상이 실수형이어야 합니다.
double result2 = (double)a / b; // 2.5
```

### 제어문과 모던 C++의 범위 기반 반복문

`if`, `switch`, `while`, 기존의 전통적인 `for`문은 기존에 알던 문법 그대로 사용하면 됩니다.

여기에 추가로, 모던 C++에서는 파이썬의 `for in`이나 자바스크립트의 `for...of`처럼 배열이나 컨테이너를 쉽게 순회할 수 있는 **범위 기반 for 루프(Range-based for loop)**가 추가되었습니다.

```cpp
int numbers[] = {1, 2, 3, 4, 5};

// 전통적인 방식
for (int i = 0; i < 5; i++) {
    std::cout << numbers[i] << " ";
}

// 모던 C++의 범위 기반 for 루프 (값 복사)
for (int num : numbers) {
    std::cout << num << " ";
}
```

데이터 구조를 다룰 때 인덱스 실수(Out of bounds)를 방지할 수 있어 실무에서 권장되는 방식입니다.

### 참조자(Reference, `&`) 형식과 범위 기반 for문 응용

C 언어의 복잡한 포인터(`*`)를 어느 정도 대체하기 위해 C++에 도입된 것이 **참조자(`&`)**입니다. 참조자는 기존 변수에 **"새로운 별명"**을 붙여주는 것과 같습니다.

```cpp
int score = 100;
int& refScore = score; // refScore는 score의 참조자 (별명)

refScore = 50; 
std::cout << score << std::endl; // 50 출력 (원본이 변경됨)
```

이러한 참조자는 **범위 기반 for문**과 결합할 때 매우 강력합니다. 단순히 `for (int num : numbers)`를 쓰면 배열의 원소를 하나씩 **"복사"**해서 가져오지만, 참조자와 타입 자동 추론(`auto&`)을 활용하면 복사 비용 없이 원본 데이터에 직접 접근하거나 수정할 수 있습니다.

```cpp
int numbers[] = {1, 2, 3};

// 참조자를 사용하여 원본 배열의 값을 2배로 증가시킴
for (auto& num : numbers) {
    num *= 2; 
} // numbers는 {2, 4, 6}이 됨

// 원본 수정을 막고 읽기 전용으로 안전하게 접근할 때는 const와 조합
for (const auto& num : numbers) {
    std::cout << num << " ";
}
```

### 형 변환 (Type Casting)

위의 나눗셈 예제에서 `(double)a`처럼 괄호를 사용하는 방식을 **C스타일 캐스팅**이라고 부릅니다. 문법이 간단해 보이지만, C++에서는 이 방식을 강력하게 지양합니다.

C 스타일 문법적으로 눈에 띄지 않아 수천 줄의 코드 속에서 찾아내기 어렵고, 컴파일러가 변환의 안전성을 제대로 검사해주지 않아 치명적인 런타임 에러를 유발할 수 있기 때문입니다.

대신 C++은 목적에 따라 명확하게 구분된 4가지 캐스트 연산자를 제공하며, 가장 기본적으로 쓰이는 것은 `static_cast`입니다.

```cpp
double pi = 3.14159;

// 나쁜 예 (C 스타일 캐스팅)
int intPi_bad = (int)pi;

// 좋은 예 (C++ 스타일 캐스팅)
int intPi_good = static_cast<int>(pi);
```

`static_cast`를 사용하면 "내가 의도적으로 실수형을 정수형으로 바꾸고 있다"는 것을 컴파일러와 다른 동료 개발자에게 명확히 알릴 수 있으며, 상식적으로 불가능한 변환(예: 포인터를 정수로 변환)을 컴파일 단계에서 차단해 줍니다.

## 배열과 구조체 기초

프로그램이 복잡해질수록 숫자 하나, 문자 하나를 따로 다루는 대신 관련된 데이터들을 하나로 묶어서 관리해야 합니다. C++에서 데이터를 묶는 가장 원초적이고 기본적인 두 가지 방법인 원시 배열(Raw Array)과 구조체(Struct)를 살펴보겠습니다.

### 원시 배열 (Raw Array)

파이썬의 리스트(List)나 자바스크립트의 배열(Array)은 크기가 알아서 늘어나고 줄어드는 아주 편리한 공간입니다. 하지만 C++의 원시 배열은 전혀 다릅니다.

```cpp
int scores[5] = {80, 90, 85, 70, 95}; // 딱 5개의 정수만 들어가는 배열
```

- **고정된 크기:** C++의 원시 배열은 한 번 크기를 정하면 프로그램 실행 중에 절대 그 크기를 바꿀 수 없습니다.
- **경계 검사가 없음 (No Bounds Checking):** 가장 위험하면서도 C++다운 특징입니다. 5개짜리 배열을 만들어 놓고 10번째 공간(`scores[9]`)에 접근하려고 하면, 자바나 파이썬은 에러를 뿜으며 프로그램을 안전하게 멈춰줍니다. 하지만 C++ 컴파일러는 아무런 경고 없이 허락되지 않은 메모리 영역을 건드리게 내버려 둡니다. 이는 치명적인 보안 취약점(버퍼 오버플로우)이나 알 수 없는 버그로 이어집니다.

> 이렇게 다루기 까다로운 원시 배열 대신, 실제 모던 C++ 실무나 알고리즘 문제 풀이에서는 크기가 자동으로 조절되고 안전한 `std::vector`라는 표준 라이브러리를 압도적으로 많이 사용합니다.

### 구조체 ( `struct` )

배열이 '같은 타입'의 데이터를 일렬로 묶는다면, 구조체는 **"서로 다른 타입"**의 데이터를 하나의 의미 있는 덩어리로 묶어줍니다.

```cpp
// 플레이어 정보를 묶어주는 구조체 정의
struct Player {
    std::string name; // 문자열
    int level;        // 정수형
    float hp;         // 실수형
};

int main() {
  // 구조체를 이용해 객체 생성
  Player p1;
  p1.name = "toran";
  p1.level = 10;
  p1.hp = 100.5f;

  std::cout << p1.name << "의 레벨: " << p1.level << std::endl;
  return 0;
}
```

- **C 언어와의 차이점:** C 언어에서의 구조체는 오직 '데이터(변수)'만 담을 수 있었습니다. 하지만 C++로 넘어오면서 구조체 안에도 함수(메서드)를 넣을 수 있게 되었습니다.

```cpp
struct UserData {
  int ID; // public
  std::string name; // public
};

class UserData2 {
  int ID; // private
  std::string name; // private
};
```

- **클래스(Class)와의 관계:** 사실 C++에서 구조체(`struct`)와 클래스(`class`)는 99% 똑같습니다. 유일한 차이점은 구조체는 내부에 선언한 변수들이 기본적으로 외부로 공개(Public)된다는 점뿐입니다. 본격적인 객체지향 설계(Phase 2)로 넘어가기 전, 단순히 데이터들을 포장해서 들고 다니는 DTO(Data Transfer Object) 역할을 할 때 구조체를 자주 사용합니다.

## 메모리 동적 할당 (`new`와 `delete`)

앞서 다룬 원시 배열은 크기가 고정되어 있다는 치명적인 단점이 있었습니다. 만약 접속하는 유저 수에 따라 유동적으로 메모리(배열이나 객체 크기)를 늘리고 싶다면 게임 실행 중(런타임)에 힙(Heap) 메모리를 빌려와야 합니다. 이를 **동적 할당(Dynamic Allocation)**이라고 합니다.

C++에서는 `new` 연산자로 메모리를 동적으로 할당받고, 다 쓴 후에는 `delete` 연산자로 시스템에 꼭 반환해야 합니다. 이는 C 언어의 `malloc` / `free`를 객체지향적으로 다루기 쉽게 발전시킨 형태입니다.

```cpp
int main() {
    // 1. 단일 변수 동적 할당
    int* pScore = new int; // int 크기의 메모리를 할당하고 그 주소를 포인터에 저장
    *pScore = 100;         // 해당 주소에 100 저장
    delete pScore;         // 빌린 메모리를 운영체제에 반납 (매우 중요!)

    // 2. 배열 동적 할당 (크기를 변수로 지정 가능)
    int size;
    std::cout << "몇 명의 점수를 저장할까요? ";
    std::cin >> size;

    int* scores = new int[size]; // 입력받은 크기만큼 배열 생성
    
    // ... scores 배열 활용 ...

    delete[] scores; // 배열을 동적 할당한 경우 삭제할 때 반드시 [] 를 붙여야 함!
    
    return 0;
}
```

> [!WARNING]
> **주의: 메모리 누수(Memory Leak)**  
> 동적으로 할당받은 메모리를 `delete`로 반납하지 않으면, 프로그램이 종료될 때까지 메모리가 할당된 채로 허공에 떠버리게 됩니다. 심해지면 메모리가 부족해 프로그램이 비정상 종료되며 이를 메모리 누수라고 부릅니다. 모던 C++에서는 이런 인적 실수를 막기 위해, 스코프를 벗어나면 자동으로 자원을 반납해 주는 **스마트 포인터(`std::unique_ptr` 등)** 사용을 적극 권장합니다.

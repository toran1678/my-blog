# \[ C++ 함수와 네임스페이스 ]

## 1. 디폴트 매개변수

C++에서는 함수를 호출할 때 인자(Argument)를 생략해도 미리 지정된 기본값이 들어가도록 설정할 수 있습니다. 이를 **디폴트 매개변수(Default Parameter)**라고 부릅니다.

### 개념 및 장점

- **정의:** 함수의 매개변수 선언 시 `= 기본값` 형태로 값을 지정해 두면, 호출 시 해당 위치의 인자를 생략할 수 있습니다.
- **장점:** 똑같은 기능을 하지만 매개변수 개수만 다른 함수를 여러 개(오버로딩) 만들 필요 없이, 하나의 함수 정의만으로 여러 호출 형태를 커버할 수 있어 코드가 아주 깔끔해집니다.

### 문법 및 예제 코드

```cpp
#include <iostream>
#include <string>

// 매개변수 price에 기본값 4500원을 지정합니다.
void orderCoffee(std::string name, int price = 4500) {
    std::cout << name << " 커피의 가격은 " << price << "원입니다." << "\n";
}

int main() {
    // 1. 모든 인자를 명시적으로 전달하는 경우 (기본값 무시됨)
    orderCoffee("바닐라 라떼", 5500); // 출력: 바닐라 라떼 커피의 가격은 5500원입니다.
    
    // 2. 두 번째 인자를 생략하는 경우 (디폴트 매개변수인 4500원이 자동으로 채워짐)
    orderCoffee("아메리카노");       // 출력: 아메리카노 커피의 가격은 4500원입니다.
    
    return 0;
}
```

> [!WARNING]
**사용 시 주의할 점**

디폴트 매개변수를 사용할 때는 다음의 두 가지 제약 사항을 꼭 지켜야 합니다.

#### 1. 디폴트 매개변수는 오른쪽부터 채우기
함수에 여러 개의 매개변수가 있을 때, **디폴트 매개변수는 반드시 맨 오른쪽부터 왼쪽으로 순서대로 빠짐없이 지정해야 합니다.**
중간에 있는 매개변수만 빼놓고 디폴트 값을 줄 수는 없습니다. 함수 호출 시 전달된 인자는 조건 없이 왼쪽부터 순서대로 매핑되기 때문입니다.

```cpp
// 잘못된 예시: 오른쪽(c)에 디폴트 값이 없는데 중간(b)에 디폴트 값을 지정함
void printData(int a, int b = 10, int c) { /* ... */ } 
// 만약 printData(1, 2)를 호출하면, 숫자 2가 매개변수 b에 들어가는지 c에 들어가는지 모호해집니다.

// 올바른 예시: 오른쪽 끝에서부터 차례대로 지정
void printData(int a, int b, int c = 20) { /* ... */ }
void printData(int a, int b = 10, int c = 20) { /* ... */ }
```

#### 2. 선언과 정의 분리 시 (.h 파일과 .cpp 파일)
C++에서는 실무적으로 헤더 파일(`.h`)에 함수를 **선언**하고, 소스 파일(`.cpp`)에 함수를 **정의**합니다.
이때 디폴트 매개변수는 **반드시 "선언부(헤더 파일)"에만 한 번 작성**해야 합니다. 정의부에 또다시 기본값을 적으면 재정의(Redefinition) 에러가 발생합니다.

* **coffee.h (선언부)**
```cpp
// 선언부에만 디폴트 매개변수를 명시하여, 호출하는 쪽에 기본값이 있음을 알립니다.
// 💡 하나의 헤더 파일에 연관된 여러 개의 함수를 선언해 모아둘 수 있습니다!
void orderCoffee(std::string name, int price = 4500);
void cancelOrder(int orderId);
```

* **coffee.cpp (정의부)**
```cpp
#include "coffee.h"
#include <iostream>

// 오류 방지를 위해 정의부에서는 기본값을 생략합니다.
// (보통 주석으로 /*= 4500*/ 와 같이 적어두는 것이 개발자 간의 좋은 매너입니다.)
void orderCoffee(std::string name, int price /*= 4500*/) {
    std::cout << name << " 커피의 가격은 " << price << "원입니다." << "\n";
}

void cancelOrder(int orderId) {
    std::cout << orderId << "번 주문이 취소되었습니다.\n";
}
```

이러한 규칙은 컴파일러가 함수 호출문 `orderCoffee("아메리카노")`를 컴파일할 때 파일 상단에 `#include` 한 **헤더 파일(선언부)**만 보고 인자가 생략되었는지를 판단하기 때문입니다!

> [!TIP]
> **굳이 `.h`와 `.cpp`를 분리하는 이유가 뭔가요? (+ 여러 개 선언하기)**
 
"그냥 `.cpp` 하나에 전부 다 적으면 안 되나요?" C++을 처음 배우면 공감하는 부분입니다.
결론부터 말하자면, 파일이 하나일 때는 안 나눠도 되지만 프로그램 규모가 조금만 커져도 **선언과 정의를 분리해야만 치명적인 에러를 막을 수 있기 때문**입니다.
 
1. **다중 정의 방지 (재사용성)**: 여러 소스 파일에서 같은 커피 함수를 쓰려고 전부 코드를 복사·붙여넣기 하면 "함수가 여러 번 정의되었다"라며 컴파일이 거부됩니다. 그래서 인터페이스인 `.h`만 여러 곳에 뿌려주고, 코드가 무거운 `.cpp`는 단 한 곳에 숨겨두어 중복을 방지합니다.
2. **무한히 모아두기 가능**: **하나의 `.h` 파일 안에 연관성 있는 여러 개의 함수를 한꺼번에 선언**해도 전혀 문제가 없습니다. 실무에서도 보통 `math_utils.h` 안에 더하기, 빼기 등 수십 개의 함수 껍데기들을 쫙 모아 둡니다.
3. **컴파일 속도 향상**: 함수 내부 동작을 살짝 수정하더라도, 껍데기는 바뀌지 않았으므로 다른 파일들은 다시 컴파일할 필요가 없어 프로젝트 빌드 성능이 압도적으로 향상됩니다!

## 2. 함수 다중 정의 (Function Overloading)

다중 정의(Overloading)는 **"이름이 같은 함수를 매개변수(인자)의 타입이나 개수만 다르게 하여 여러 개 정의하는 문법"**입니다. C언어에서는 불가능했지만 C++로 넘어오면서 가능해진 객체지향적 특징 중 하나입니다.

### 개념 및 장점

- **정의:** 함수 이름이 같더라도, 전달받는 인자의 자료형(`int`, `double`, `string` 등)이나 개수가 다르다면 컴파일러가 이를 서로 다른 함수로 인식합니다.
- **장점:** 비슷한 기능을 하는 함수들의 이름을 통일할 수 있어 이름 짓기(Naming)의 고통이 줄어들고, 사용자 입장에서도 일관된 함수명 하나만 기억하면 되므로 사용성이 크게 향상됩니다.

### 문법 및 예제 코드

```cpp
#include <iostream>

// 1. 정수 두 개를 더하는 함수
int add(int a, int b) {
    return a + b;
}

// 2. 실수 두 개를 더하는 함수 (이름은 같지만 매개변수 타입이 다름)
double add(double a, double b) {
    return a + b;
}

// 3. 세 개의 정수를 더하는 함수 (이름은 같지만 매개변수 개수가 다름)
int add(int a, int b, int c) {
    return a + b + c;
}

int main() {
    std::cout << add(10, 20) << "\n";          // 1번 함수 호출 (출력: 30)
    std::cout << add(3.14, 1.59) << "\n";      // 2번 함수 호출 (출력: 4.73)
    std::cout << add(1, 2, 3) << "\n";         // 3번 함수 호출 (출력: 6)
    return 0;
}
```

### 사용 시 주의할 점

함수 다중 정의와 앞서 배운 **디폴트 매개변수**를 같이 사용할 때는 컴파일러가 어떤 함수를 호출해야 할지 헷갈리는 **모호함(Ambiguity)**이 발생할 수 있습니다.

```cpp
void printNumber(int a) { /* ... */ }
void printNumber(int a, int b = 10) { /* ... */ }

// main 함수에서 호출 시
// printNumber(5); // 컴파일 에러 발생. 첫 번째 함수인지, 두 번째 함수의 b 생략 버젼인지 모호함
```

## 3. 인라인 함수 (Inline Function)

인라인 함수는 이름 그대로 **코드의 라인(line) 안(in)으로 쏙 들어가는 함수**를 의미합니다. 함수 호출로 인한 성능 저하(오버헤드)를 막고 실행 속도를 극대화하기 위해 C++에서 제공하는 키워드입니다.

### 동작 원리

보통 함수를 호출하면, 프로그램은 실행하던 위치를 기억해 두고 함수의 메모리 주소로 점프(Jump)하여 코드를 실행한 뒤 다시 원래 위치로 돌아옵니다. 이 점프 과정에서 미세한 시간이 소모됩니다(함수 호출 오버헤드).  
`inline` 키워드를 붙이면 컴파일러는 함수로 점프하는 대신, 함수를 호출한 자리에 함수의 실제 코드를 **통째로 복사해서 그대로 붙여넣습니다.**

### 문법 및 예제 코드

```cpp
#include <iostream>

// inline 키워드를 붙여 매크로처럼 빠른 동작을 유도합니다.
inline int square(int x) {
    return x * x;
}

int main() {
    int result = square(5); 
    // 실제 컴파일 시점에는 아래와 같이 복사-붙여넣기 된 것처럼 치환됩니다.
    // int result = 5 * 5; 
    
    std::cout << result << "\n";
    return 0;
}
```

### 사용 시 주의할 점

1. **길이가 짧은 함수에만 사용**: 코드가 긴 함수를 인라인으로 만들면 어떻게 될까요? 여기저기서 호출할 때마다 거대한 코드가 계속 복사되므로, 실행 파일(exe)의 크기가 비정상적으로 뚱뚱해지는 심각한 부작용(Code Bloat)이 발생합니다. 따라서 주로 1~3줄짜리 짧은 계산식이나 데이터를 넣고 빼는 `C++ 클래스`의 getter/setter 함수에만 사용하는 것이 원칙입니다.
2. **최종 결정은 컴파일러가**: 개발자가 명시적으로 `inline`을 적어주더라도, 컴파일러가 보기에 코드가 너무 길거나 `for`, `while`문처럼 복잡하면 자체적으로 인라인 처리를 무시하고 일반 함수처럼 점프하도록 만듭니다. 반대로 개발자가 적지 않아도 알아서 인라인 처리해 버리기도 합니다. (현대 스마트 컴파일러의 최적화 기능)

## 4. 네임스페이스 (Namespace)

프로젝트 규모가 커지고 여러 개발자가 협업하거나 외부 라이브러리를 가져다 쓸 때 가장 많이 발생하는 문제는 **'이름 충돌(Name Collision)'**입니다. 내 코드에도 `print()` 함수가 있고 라이브러리에도 `print()` 함수가 있다면 빌드 에러가 발생합니다.
이를 방지하기 위해 **"함수나 변수들이 소속되는 가상의 울타리(소속명)"**를 만들어주는 문법이 바로 네임스페이스(Namespace)입니다. 성(姓)이 같아도 소속된 팀이 다르면 서로를 구분할 수 있는 것과 같습니다.

### 문법 및 예제 코드

```cpp
#include <iostream>

// Graphic 팀의 네임스페이스
namespace Graphic {
    void init() {
        std::cout << "그래픽 시스템 초기화\n";
    }
}

// Network 팀의 네임스페이스
namespace Network {
    void init() {
        std::cout << "네트워크 시스템 초기화\n";
    }
}

int main() {
    // 소속(::)을 명시하여 이름이 같은 코드를 충돌 없이 호출합니다.
    Graphic::init(); 
    Network::init(); 
    return 0;
}
```

### `using namespace`와 실무 가이드라인

매번 `Graphic::`, `std::` 처럼 네임스페이스와 범위 지정 연산자(`::`)를 적어주기가 귀찮을 때 **`using` 지시어**를 흔히 사용합니다. 
(예: `using namespace std;`) 하지만 실무에서는 다음과 같은 이유로 이 구문을 아주 엄격히 통제합니다.

1. **`using namespace` 전체 범위 선언 지양**: 헤더 파일이나 전역 공간 상단에 선언해 버리면, 기껏 만들어 둔 네임스페이스의 울타리가 전부 무너져 이름 충돌 위험이 다시 커집니다.
2. **좁은 지역에서만 사용**: 꼭 필요하다면 특정 함수 내부나 분리된 블록(`{}`) 안에서만 사용하는 것이 안전합니다.
3. **특정 요소만 콕 집어 가져오기**: 전체 공간을 여는 대신 `using std::cout;`, `using std::string;` 처럼 사용할 함수나 객체 몇 가지만 명시적으로 스코프에 추가하는 것이 좋은 관례입니다.

## 5. 식별자 검색 순서 (Identifier Search Order)

우리가 C++에서 어떤 함수나 변수의 이름(식별자)을 코드에 적으면, 컴파일러는 도대체 어떤 대상을 써야 할지 **정해진 순서(우선순위)대로 공간을 검색**하며 알맞은 녀석을 골라냅니다.

만약 이름 앞에 범위 지정 연산자(`::`) 없이 이름만 적혀 있다면, 컴파일러는 다음 순서로 탐색을 진행합니다.

1. **현재 블록(Local Scope)**: 가장 먼저 탐색하며, 현재 줄이 위치한 중괄호 `{ }` 안(지역 변수)을 찾아봅니다. (이를 이름 가림, Name Hiding 현상이라고도 합니다.)
2. **둘러싼 외부 블록**: 만약 블록이 중첩되어 있다면 내부에서부터 순차적으로 바깥 블록으로 올라가며 탐색합니다.
3. **클래스 멤버 공간**: 현재 클래스 내부를 살펴봅니다 (객체지향 설계 적용 시).
4. **전역 공간(Global) 및 네임스페이스**: 파일 레벨이나 `using`으로 전역 공간에 노출된 네임스페이스 안을 찾아봅니다.

```cpp
#include <iostream>

int number = 100; // 4순위 (전역 공간 선언 변수)

namespace MySpace {
    int number = 50;
}

int main() {
    // 1순위 (현재 main 블록 안의 지역 변수)
    int number = 10; 
    
    // 순차 탐색 규칙에 의해 가장 가까운 지역 변수인 10을 먼저 채택하여 출력합니다.
    std::cout << "기본 검색: " << number << "\n"; 
    
    // 10 때문에 가려진 전역 변수인 100을 꺼내고 싶다면?
    // 범위 지정 연산자(::) 앞에 아무것도 안 적으면 전역 공간을 의미합니다.
    std::cout << "전역 변수 검색: " << ::number << "\n"; 
    
    // 네임스페이스 안의 50을 꺼내고 싶다면?
    std::cout << "네임스페이스 검색: " << MySpace::number << "\n";
    
    return 0;
}
```

이렇듯 모호함을 배제하는 명확한 검색 규칙과 범위 지정(`#include`, `.h` 분리, `namespace`) 덕분에, 수백만 줄에 달하는 대규모 C++ 프로젝트에서도 방대한 코드들이 엉키지 않고 질서 있게 동작할 수 있습니다.
